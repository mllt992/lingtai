## 2. 代码质量

### 2.1 正确性
- [ ] 主流程跑通
- [ ] 至少一条异常路径跑通
- [ ] 边界条件：空 / null / 空数组 / 零 / 负数 / 超大 / 超长 已考虑
- [ ] 验收标准逐条满足

### 2.2 规范
- [ ] 命名、代码风格符合 `01_命名与代码风格规范.md`
- [ ] API 设计符合 `02_API接口设计规范.md`
- [ ] 数据库变更符合 `03_数据库设计规范.md`
- [ ] 前端代码符合 `04_前端开发规范.md`
- [ ] 后端代码符合 `05_后端开发规范.md`
- [ ] 编译 / 类型检查 / Lint 0 错误（警告有豁免理由）

### 2.3 可维护性
- [ ] 函数 / 类单一职责
- [ ] 无 console.log / 调试代码 / 注释掉的代码
- [ ] 无遗留 TODO（或 TODO 有负责人）
- [ ] 无魔法值（提取常量或写明含义）

---

## 3. 数据与边界

- [ ] **日期范围**：开闭区间在 SQL / 后端 / 前端三端一致
- [ ] **时区**：传输 ISO-8601，转换在边界层一次性完成
- [ ] **小数 ↔ 百分比**：转换只发生一次
- [ ] **金额**：高精度类型 / 整数（分），千分位 + 等宽字体 + 右对齐
- [ ] **分页**：page 起始一致，`total/hasNext` 有断言
- [ ] **排序**：默认排序字段稳定，带 ID tiebreaker
- [ ] **空值**：null / "" / 0 / [] 行为明确，UI 不出现 "undefined" / "null"
- [ ] **超长**：文本输入达上限不报 500，列表显示有截断 + hover 全文
- [ ] **大数据量**：列表/导出有分页或虚拟滚动

---

## 4. 权限与安全

- [ ] 每个新增接口都做了**功能权限**校验
- [ ] 每个查询都做了**数据权限**（行级过滤在 SQL 层）
- [ ] 列表过滤了，详情 / 导出 / 更新 / 删除接口同样过滤
- [ ] **越权场景测过**：构造非所有者请求 → 403 / 不可见
- [ ] **输入校验**在后端（不能仅前端）
- [ ] **SQL 参数化**，无字符串拼接
- [ ] 敏感字段（密码 / token / 身份证 / 手机号 / 银行卡）：存储 hash/加密，日志脱敏，响应脱敏
- [ ] 文件上传：类型白名单 + 大小限制 + 重命名存储

---

## 5. 并发与一致性

- [ ] **写入**：有乐观锁或唯一索引，并发提交一成一败
- [ ] **防重**：按钮 loading + 后端幂等
- [ ] **事务**：跨表写入在同一事务，**外部调用不卷入事务**
- [ ] **缓存**：失效策略明确，与 DB 一致
- [ ] **消息消费者幂等**

---

## 6. 错误处理与日志

- [ ] 业务异常 → 4xx + 业务码 + 友好消息
- [ ] 系统异常 → 5xx + traceId + 不暴露堆栈给前端
- [ ] **无空 catch**、**无吞错误**
- [ ] 关键业务事件有 INFO 日志（创建、状态流转、登录登出）
- [ ] 异常路径有 WARN / ERROR
- [ ] ERROR 日志带完整堆栈 + 业务上下文（traceId、userId、业务 ID）
- [ ] 日志**无明文敏感字段**

---

## 7. 前端交互

- [ ] **四态齐全**：加载 / 空 / 错误 / 权限
- [ ] **响应式**：常见分辨率不破版
- [ ] **可访问性**：键盘导航、焦点可见、对比度、aria
- [ ] 视觉风格走设计系统，无硬编码颜色字号
- [ ] 浏览器 Console 干净：无红错、无未捕获 Promise rejection
- [ ] Network 面板正常路径无 4xx / 5xx
- [ ] 表单提交防重、错误回显到字段

---

## 8. 后端实现

- [ ] Controller 薄，无业务逻辑
- [ ] Service 方法对应一个业务用例
- [ ] Repository 纯 CRUD
- [ ] DTO 与 Entity 分离
- [ ] 外部调用有超时和重试
- [ ] 关键查询有索引，无 N+1
