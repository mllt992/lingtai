# 凌台 · Loft 实施方案

- 文档版本：v1.0
- 编写日期：2026-05-13
- 适用范围：Windows 11 桌面端（macOS/Linux 复用，本期 Windows 优先）

---

## 一、产品定位

**凌台（Loft）** 是一款驻留 Windows 桌面的轻量级控制台，聚合"快速启动 + 文件归纳 + 端口监控 + 性能监控"四大场景，目标是让用户在一个窗口内完成日常桌面管理与开发自检。

**核心原则**：低占用、原生性能、现代美观、可定制。

## 二、功能矩阵

| 模块 | 子功能 | 来源 |
|---|---|---|
| 启动器 Launcher | 自动扫描开始菜单快捷方式 | `Public\Start Menu` + `AppData\Roaming\Microsoft\Windows\Start Menu` 递归 `.lnk` |
| | 手动添加（输入路径 / 拖拽文件 / 拖拽 URL） | Tauri drop event + 表单 |
| | 分组管理 / 收藏 / 自定义图标 | 本地 JSON |
| | 一键启动（双击 / 回车） | `std::process::Command` |
| | 搜索过滤 | 前端 fuzzy 匹配 |
| 资源归纳 Resources | 文件夹快捷打开 | `shell::open` |
| | 文件快捷打开 | `shell::open` |
| | 网站快捷启动 | `shell::open` 默认浏览器 |
| | 三类统一卡片视图，按标签分组 | 同 JSON 数据源 |
| 端口监控 Ports | 用户进程监听端口列表 | `netstat2` 枚举 + `sysinfo` 反查进程名 |
| | 系统级 PID 自动过滤（PID < 1000 且属于系统进程） | 白名单 + 黑名单 |
| | 一键终止占用进程 | `sysinfo::Process::kill` |
| | 自动刷新（默认 5s） | 前端定时 invoke |
| 性能监控 Monitor | CPU 总体使用率 + 各核心 | `sysinfo` |
| | 内存 / Swap 已用与总量 | `sysinfo` |
| | 磁盘各盘符使用率 | `sysinfo::Disks` |
| | GPU 名称 / 显存 / 使用率（Nvidia 优先，其它兜底） | `nvml-wrapper`，失败显示"N/A" |
| | 60 秒滚动 SVG 折线图 | 前端轻量 SVG |
| 设置 Settings | 主题切换（6 预设 + 自定义主色） | CSS 变量 |
| | 启动器扫描路径配置 | JSON |
| | 端口刷新间隔 / 是否过滤系统进程 | JSON |
| | 开机自启 / 最小化到托盘 | Tauri autostart plugin + tray |
| | 数据导入 / 导出 | 文件对话框 |

## 三、技术架构

```
┌─────────────────────────────────────────┐
│ Vue 3 + TypeScript + UnoCSS + Pinia     │  前端
│   - Views: Launcher / Resources /       │
│            Ports / Monitor / Settings   │
│   - Stores: launcher / monitor / ports  │
│            settings                     │
│   - Theme: CSS Custom Properties        │
└──────────────┬──────────────────────────┘
               │ invoke()
┌──────────────┴──────────────────────────┐
│ Tauri 2 IPC Bridge                      │
└──────────────┬──────────────────────────┘
               │ #[tauri::command]
┌──────────────┴──────────────────────────┐
│ Rust Backend                            │
│   commands/launcher.rs  — scan/launch   │
│   commands/files.rs     — open/dialog   │
│   commands/monitor.rs   — sysinfo/nvml  │
│   commands/ports.rs     — netstat2      │
│   commands/settings.rs  — load/save     │
│                                         │
│   crates: sysinfo, netstat2,            │
│          nvml-wrapper, lnk,             │
│          serde, tokio, walkdir          │
└─────────────────────────────────────────┘
```

### 关键依赖

- `tauri = "2"` — 应用框架
- `sysinfo = "0.32"` — CPU/RAM/Disk/Process（纯 Rust，跨平台）
- `netstat2 = "0.9"` — 端口枚举
- `nvml-wrapper = "0.10"` — NVIDIA GPU（可选）
- `lnk = "0.5"` — 解析 Windows 快捷方式
- `walkdir = "2"` — 目录递归
- `serde / serde_json` — 数据序列化
- `tokio` — 异步运行时

### 性能预算

| 指标 | 目标 |
|---|---|

| 空闲内存占用 | ≤ 60 MB |

## 四、UI 设计要点

- **布局**：左侧 72px 图标导航 + 主区域。无系统标题栏，自绘 32px 顶栏（拖动 + 最小化/最大化/关闭）。
- **视觉**：圆角 12-16px、毛玻璃顶栏、卡片悬浮微阴影、统一 4 / 8 / 12 / 16 / 24 间距栅格。
- **动效**：路由切换 fade-slide 180ms；卡片 hover scale 1.02。
- **主题**：通过 `<html data-theme="aurora|carbon|vanilla|nord|solar|mint">` 切换；所有颜色走 CSS 变量；用户可在设置中覆盖主色（`--accent`）。

## 五、数据持久化

存放在 Tauri 应用数据目录（Windows: `%APPDATA%\loft\`），结构：

```
loft/
├── config.json          # 用户设置
├── launcher.json        # 用户添加的启动项
└── resources.json       # 用户添加的文件/文件夹/网址
```

JSON 结构示例：

```json
{
  "version": 1,
  "theme": "aurora",
  "accent": "#5b8cff",
  "scanPaths": ["%ALLUSERSPROFILE%\\Microsoft\\Windows\\Start Menu\\Programs"],
  "ports": { "refreshMs": 5000, "filterSystem": true },
  "monitor": { "refreshMs": 1000, "history": 60 }
}
```

## 七、风险与对策

| 风险 | 影响 | 对策 |
|---|---|---|
| GPU 监控异厂商兼容 | 信息缺失 | NVML 优先，非 N 卡显示"暂不支持"，后续按需扩 WMI |
| 端口枚举需管理员权限 | 进程名缺失 | 列出端口但标注"权限不足"，提供"以管理员重启"按钮 |
| 快捷方式解析失败 | 启动项缺失 | 兜底直接取 `.lnk` 文件名，记录日志不抛错 |
| Tauri 拖拽事件冲突 | 拖入文件被浏览器拦截 | 用 Tauri `onDragDropEvent` 而非 HTML5 DnD |
| 中文路径 / 特殊字符 | 启动失败 | 一律走系统 shell 调用，避免手拼命令行 |
