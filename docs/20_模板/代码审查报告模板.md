## 1. 审查概要

- **整体结论**：✅ 通过 / ⚠️ 修复后通过 / ❌ 不通过
- **必修问题**：N 项（详见下方）

---

## 2. 维度一：正确性

### 业务正确
- [ ] 完整实现需求/方案的全部内容
- [ ] 验收标准逐条满足
- [ ] 边界条件处理：空 / null / 空数组 / 零 / 负数 / 超大 / 超长
- [ ] 异常路径处理：超时、并发、依赖失败、权限不足
- [ ] 同一业务规则在多处表达一致

### 代码正确
- [ ] 类型 / 空指针 / 除零 / 越界
- [ ] 无死循环 / 死锁 / 无限递归
- [ ] 资源释放完整（文件、连接、订阅、监听）
- [ ] 数值精度（金额、百分比、日期）

### 数据正确
- [ ] 事务边界正确，外部调用未卷入 DB 事务
- [ ] 软删除字段全部生效
- [ ] 业务唯一性在 DB 层有保护
- [ ] 并发安全（乐观锁 / 唯一索引 / 防重）

### 发现的问题
| 编号 | 严重 | 位置 | 问题描述 | 建议 |
|---|---|---|---|---|
| 1 | 阻断 | xxx.java:42 | 未处理 null 导致 NPE | 加 null 校验 |

## 5. 维度四：安全性

对照 `docs/00_通用规范/06_安全规范.md`：

- [ ] 接口要求登录（白名单已显式标注）
- [ ] 功能权限在后端校验
- [ ] 数据权限在 SQL 层过滤
- [ ] 列表 / 详情 / 导出 / 更新 / 删除均做了权限
- [ ] 越权场景已测试
- [ ] 输入校验在后端
- [ ] SQL 用参数化，无字符串拼接
- [ ] 敏感字段（密码 / token / 身份证 / 手机号 / 银行卡）：存储 hash/加密，日志脱敏，响应脱敏
- [ ] 依赖来源可信，无已知漏洞
- [ ] 文件上传校验类型 / 大小 / 内容
